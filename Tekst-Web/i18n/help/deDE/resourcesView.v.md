# Ressourcen-Übersicht

Diese Übersicht zeigt alle verfügbaren Informationen zu den Ressourcen des aktuellen Arbeitstextes zentral an. Je nachdem, ob sie mit einem Konto angemeldet sind und welche Berechtigungen Sie haben, sind hier unterschiedliche Funktionen für Sie verfügbar.

## Filter

Mit dem Filter-Eingabefeld können Sie Ressourcen nach einer Vielzahl von Kriterien filtern. Geben Sie dazu einfach die gewünschten Werte in das Feld ein. Sie werden dann nach Leerzeichen und Kommata getrennt und einzeln ausgewertet.

## Aktionen

Das letzte Element in jeder Ressourcen-Information ist die Aktionsleiste. Hier können Sie je nach Berechtigung verschiedene Aktionen ausführen, die im Folgenden kurz erläutert sind. Der Buchstabe in Klammern am Ende der Überschrift gibt jeweils an, welche Nutzer\*innenrolle für die jeweilige Aktion mindestens erforderlich ist. Dabei steht `v` für einfache Besucher, `u` für registrierte und angemeldete Nutzerinnen, `o` für Besitzer\*innen der jeweiligen Ressource und `s` für Administrator\*innen (Superuser) der Plattform.

### Lesen (`v`)

Zeigt diese Ressource isoliert in der Leseansicht an.

### Version erstellen (`u`)

Wenn Sie die Absicht haben, eine größere Menge Änderungen oder Ergänzungen an einer bestehenden Ressource vorzuschlagen, ist eine Ressourcen-Version ein guter Weg, um diese Änderungen zu sammeln und dann als Ganzes zu übermitteln. Die Besitzer\*in der Originalressource kann so besonders leicht Ihre Vorschläge einzeln prüfen und gegebenenfalls übernehmen.

Für vereinzelte Änderungen ist es einfacher, in der Leseansicht am entsprechenden Inhalt direkt die Funktion "Korrekturvorschlag" zu nutzen.

### Einstellungen (`o`)

Öffnet die Einstellungen für diese Ressource. Dort finden sie gesonderte Informationen und Hilfetexte.

### Löschen (`o`)

Löscht eine Ressource inklusive aller Inhalte vollständig und unwiderruflich. Bitte nutzen Sie diese Funktion mit Bedacht und veranlassen Sie ggf. vorab eine Sicherung der Daten!

### Export (`v`)

Öffnet das Exportwerkzeug für diese Ressource. Dies entspricht der Exportfunktion in der Leseansicht.

### Korrekturnotizen anzeigen (`o`)

Falls für diese Ressource Korrekturnotizen von anderen Nutzer\*innen vorliegen, können Sie diese hier einsehen, verwalten, prüfen und ggf. verarbeiten.

### Inhalte bearbeiten (`o`)

Öffnet die Bearbeitungsansicht für Inhalte dieser Ressource. Hier werden sie gesonderte Informationen und Hilfetexte finden.

### Importvorlage herunterladen (`o`)

Lädt eine JSON-basierte Vorlage für einen Daten-Import in diese Ressource herunter. Damit der Import erfolgreich durchgeführt werden kann, müssen die in die Vorlage eingefügten Daten dem Schema entsprechen, das ebenfalls mit der Vorlage ausgeliefert wird.

### Vorbereitete Inhalte importieren (`o`)

Wenn Sie eine vorbereitete JSON-Datei mit Inhalten für diese Ressource haben, können Sie diese hier hochladen. Dabei werden neue Inhalte auf Basis der Import-Daten neu angelegt und bestehende Inhalte überschrieben.

### Für Veröffentlichung vorschlagen (`o`)

Indem Sie eine von Ihnen erstellte Ressource zur Veröffentlichung vorschlagen, stellen Sie sie allen registrierten Benutzer\*innen der Plattform zur Verfügung, damit diese sie begutachten und Änderungen oder Ergänzungen vorschlagen können. Auch die Administrator\*innen der Plattform werden über Ihren Vorschlag informiert. Diese können, sofern Interesse besteht und sobald sie mit der Qualität der Daten zufrieden sind, ihre Ressource auf der Plattform veröffentlichen. Dadurch wird sie der Öffentlichkeit – also auch nicht registrierten Benutzer\*innen – zugänglich gemacht.

Ressourcen, die zur Veröffentlichung vorgeschlagen sind, können nicht editiert werden. Außerdem werden jegliche anderen Benutzer\*innen erteilten Zugriffsberechtigungen zurückgesetzt. Die Zeit, in der eine Ressource "vorgeschlagen" ist, dient der Begutachtung durch die anderen Benutzer\*innen und dem Sammeln von Änderungsvorschlägen. Um die Vorschläge einzuarbeiten, muss der Vorschlag zunächst zurückgezogen werden. Ist die Ressource dann überarbeitet, kann sie erneut vorgeschlagen werden. Ein Vorschlag zur Veröffentlichung kann zurückgezogen werden, solange die Ressource nicht veröffentlicht wurde.

Wird eine Ressource veröffentlicht, erhalten die Plattformadministrator\*innen zunächst die alleinige Kontrolle über sie, um unerwünschte Änderungen zu verhindern. Sie können auch später noch Änderungen an den Daten der Ressource vorschlagen, diese müssen jedoch gesondert genehmigt werden, es sei denn die Administrator\*innen fügen Sie wieder als Besitzer\*in der Ressource hinzu.

### Veröffentlichungsvorschlag widerrufen (`o`)

Zieht den Vorschlag zur Veröffentlichung einer Ressource zurück. Dadurch ist sie wieder nur für ihre Besitzer\*innen sichtbar und kann "privat" überarbeitet werden.

### Veröffentlichen (`s`)

Eine Ressource kann nur veröffentlicht werden, wenn sie vorher zur Veröffentlichung vorgeschlagen wurde. Die Plattformadministrator\*innen erhalten dann vorerst die alleinige Kontrolle über die Ressource und sie wird der Öffentlichkeit – also auch nicht registrierten Nutzer\*innen – zugänglich gemacht.

### Veröffentlichung aufheben (`o`)

Macht eine Veröffentlichug wieder rückgängig. Die Ressource ist dann wieder nur für ihre Besitzer\*innen sichtbar.

### Ressourcen-Besitzer\*innen festlegen (`o`)

Mit dieser Funktion lassen sich die Besitzer\*innen einer Ressource ändern. Die neuen Besitzer\*innen haben dann Schreibzugriff auf die Inhalte der Ressource, können Einstellungen ändern und die Ressource allgemein verwalten, also auch selbst weitere Besitzer\*innen hinzufügen.

Legen Sie deshalb die Besitzer\*innen einer Ressource nur fest, wenn Sie sicher sind, dass diese die Verantwortung für die Ressource übernehmen können und Sie ihnen vertrauen!
